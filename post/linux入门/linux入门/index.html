linux入门

linux文件结构

典型的linux目录如下：

/
├── bin bin是Binary（二进制程序）的缩写, 这个目录存放着最经常使用的命令
├── boot 
├── dev dev是Device(设备)的缩写, 该目录下存放的是Linux的外部设备，在Linux中访问设备的方式和访问文件的方式是相同的。
├── etc 这个目录用来存放所有的系统管理所需要的配置文件和子目录。
├── home 用户目录
├── proc
├── root
├── run
├── sbin
├── srv
├── sys
├── tmp 临时目录，通常是tmpfs，该目录中的文件实际上存储在内存中
├── usr
└── var

linux中目录是以树的结构出现的，所有目录都是以/根目录为根的一颗树

一切皆文件的思想

一切皆文件的思想是指几乎所有（目录、字符设备、块设备、套接字、打印机、进程、线程、管道等）都被映射到了linux文件树上，成为一个树上的一个节点。

举个例子：

/dev/sda  这是我的硬盘，在linux下被映射为了sd(x)
/dev/sda1 这是上述硬盘的第一个分区
/dev/sda2 这是上述硬盘的第二个分区

linux文件权限

linux是一个多用户多任务的操作系统，这意味着一个linux系统可以有多个用户同时登录并且同时执行多个任务。那么文件权限就是一个很重要的功能。

使用ls -l命令

我们以图中sh.c 为例

sh.c的权限为- rw- r-- r--(我们将它拆成四个部分,第一部分一个字符,其余部分3个字符)

第一部分的字符指定了该文件的类别,文件类型主要有以下几种:
- 普通文件
d 目录
l 链接文件
b 块特殊设备
c 特殊设备文件 
s 套接字文件
p 管道文件

其余三个部分代表了文件的权限:
主人的权限                   rw-
用户组(文件的拥有组)的权限     r--
其他人的权限                 r--

其中 r代表read 可读;w代表write 可写;x代表可执行

同时 文件权限也可以用数字(3个二进制值)来表示 权限|值 –|:–: r|4| w|2| x|1| 比如上述权限可表示为 644 (4+2)(4)(4)

简单使用chmod修改文件权限(简单点使用数字模式)

chmod 666 sh.c
-rw-rw-rw-  1 andy andy  6129  1月 18 22:00 sh.c

shell简介

Unix shell，一种壳层与命令行界面，是UNIX操作系统下传统的用户和计算机的交互界面。用户直接输入命令来执行各种各样的任务。

cd命令

当一个终端仿真器启动时，会默认工作目录在启动它的用户的主目录/home/userName，通常被简写为~。通过cd 命令，我们可以切换当前目录。

cd / 这个命令会让我们切换到根目录

ls命令

ls命令用于列出某个目录下的文件或目录（当然还有很多扩展用途）

ls -a 显示所有类型的文件或目录

特殊目录 .和..

.目录代表当前工作目录，..代表当前工作目录的父目录

man

man是一个很重要的文档库，里面拥有丰富的帮助文档。man中几乎包含你所需要的所有文档。具体包括如下章节：

  1. 可执行程序或 shell 命令
  2. 系统调用(内核提供的函数)
  3. 库调用(程序库中的函数)
  4. 特殊文件(通常位于 /dev)
  5. File formats and conventions, e.g. /etc/passwd
  6. 游戏（甚至有游戏教程）
  7. 杂项(包括宏包和规范，如 man(7)，groff(7))
  8. 系统管理命令(通常只针对 root 用户)
  9. 内核例程 [非标准

使用man

man man #man （命令名）

上图就是执行该命令的结果,一般会在

小节名称通常包括

  • NAME
  • 概述(SYNOPSIS)
  • 配置(CONFIGURATION)
  • 描述(DESCRIPTION)
  • 选项(OPTIONS)
  • 退出状态(EXIT STATUS)
  • 返回值(RETURN VALUE)
  • 错误(ERRORS)
  • 环境(ENVIRONMENT),
  • 文件(FILES)
  • 版本(VERSIONS)
  • 符合标准(CONFORMING TO)
  • 注(NOTES)
  • 缺陷(BUGS)
  • 示例(EXAMPLE)
  • 作者(AUTHORS)
  • 亦见(SEE ALSO).

命令格式

一般命令的使用方法出现在概述(SYNOPSIS,选项(OPTIONS)和示例(EXAMPLE)三个部分,格式如下:

  • 加粗文本 按原样显示。
  • 倾斜文本 用相应的参数进行替换。
  • [-abc] “ ” 内的任意/全部参数都是可选的。
  • -a|-b 以“|”分隔的选项可以一起使用。
  • 参数 … 参数 可以重复。
  • [表达式] … “ ”内的整个 表达式 可以重复。

man的一般使用方式

man ls
显示 项目 (程序)  ls 对应的手册页

man man.7
Display the manual page for macro package man from section 7.  (This is an alternative
spelling of "man 7 man".)

man 'man(7)'
Display the manual page for macro package  man  from  section  7.   (This  is  another alternative  spelling  of  "man  7  man".   It may be more convenient when copying and pasting cross-references to manual pages.  Note that the parentheses must normally  be quoted to protect them from the shell.)

man -a intro
正确执行时显示手册中包含的所有可用的  intro 手册页。可以在连续显示过程中退出或跳过任一
页。

man -t bash | lpr -Pps
Format the manual page for bash into the default troff or groff format and pipe it  to the printer named ps.  The default output for groff is usually PostScript.  man --help should advise as to which processor is bound to the -t option.

如果man没有你使用的命令的文档怎么办

  • -h 显示帮助（不一定是这个功能）
  • –help 显示帮助
  • –version 显示版本
